# コンプライアンス・ナレッジベース検索（週次課題プロトタイプ）

条文IDで紐づけた法令データを、キーワードで検索できるシンプルな検索ツールです。
PHP / HTML / CSS / JavaScript の週次課題として作成したものですが、将来的な事業構想
「AI活用による不動産投資運用の自動化支援」における MVP（公開情報を条文IDで紐づけたナレッジベース）
の最初の一歩としても位置づけています。

## できること

1. キーワード（またはタグ）を入力すると、ローカルのナレッジベース（`data/laws.json`）の中から
   関連する条文を検索する
2. ヒットした条文のうち1件について、デジタル庁が提供する公式の **e-Gov法令API** から
   正式な条文原文をリアルタイムで取得し、あわせて表示する
3. e-Govの該当条文ページへ直接飛べるリンクを表示する

対象データは「不動産・宅建」（宅地建物取引業法）と「金融・コンプライアンス」（金融商品取引法）の
2テーマ、計8条文を収録しています。

## ファイル構成

役割ごとにファイルを分割しています。手直しの際に該当ファイルだけを開けば済むようにする狙いです。

| ファイル | 役割 |
|---|---|
| `index.html` | 画面のマークアップのみ（見た目・振る舞いは含まない） |
| `style.css` | 見た目（配色・レイアウト） |
| `app.js` | 振る舞い（入力チェック、`search.php`への非同期リクエスト、結果の描画） |
| `search.php` | 検索ロジック本体。ローカルJSON検索 ＋ e-Gov法令APIとの連携 |
| `data/laws.json` | ナレッジベースの実データ（条文ID・法令名・タグ・要約） |

```
project/
├── index.html
├── style.css
├── app.js
├── search.php
└── data/
    └── laws.json
```

アップロード・配置の際は、この階層構造（特に `data/` フォルダ）を崩さないでください。
`search.php` は `__DIR__ . '/data/laws.json'` という相対パスで読み込んでいるためです。

## 動かし方

### ローカル（XAMPP / MAMP）

1. `htdocs`（XAMPPの場合）配下に、このフォルダ一式をそのままコピーする
2. Apache（とPHP）を起動する
3. ブラウザで `http://localhost/（コピーしたフォルダ名）/` を開く

### さくらインターネットのレンタルサーバー

1. FTP等で、フォルダ構造を保ったままサーバー上の公開ディレクトリにアップロードする
2. ブラウザで設置先のURLにアクセスする

`search.php` は `file_get_contents` が使えない環境（`allow_url_fopen = Off`）でも、
自動的に `cURL` にフォールバックして e-Gov法令API を呼び出すようにしているため、
サーバー側の設定差異を気にせず動作するはずです。

## データの追加・編集方法

`data/laws.json` に、以下の形式でオブジェクトを追加するだけで検索対象が増えます。

```json
{
  "id": "法令ID-条番号",
  "law_name": "法令の正式名称",
  "law_id": "e-Gov上の法令ID",
  "article_num": "条番号（数字のみ）",
  "article_label": "第◯条",
  "category": "分類ラベル（任意）",
  "tags": ["検索用タグ1", "検索用タグ2"],
  "summary": "条文の要約文"
}
```

`law_id` は e-Gov法令検索（https://laws.e-gov.go.jp/）で対象の法令を開いたときのURLから確認できます。

## 検索の仕組み（中身）

1. `app.js` がキーワードを `search.php?keyword=...` へ送る
2. `search.php` が `data/laws.json` を読み込み、`summary` / `article_label` / `tags` を対象に
   部分一致検索する（PHPの `mb_stripos` を使用。キーワードベースのルール検索）
3. ヒットした最初の1件について、`law_id` と `article_num` を使って e-Gov法令API
   （`https://laws.e-gov.go.jp/api/1/articles;lawId=...;article=...`）にリクエストし、
   返ってきたXMLから条文本文（`Sentence`要素）を抽出する
4. 結果をJSONで `app.js` に返し、画面に描画する

## 現時点でのスコープと、今後の発展方向

これは**ルールベースのキーワード検索**であり、AIエージェントではありません。
入力されたキーワードと、事前に用意したタグ・要約文が一致するかどうかだけを見ています。

将来的に発展させる場合の方向性（事業構想のロードマップに対応）：

- **Step1（今回）**：公開法令を条文IDで紐づけたナレッジベースの検索基盤
- **Step2**：自社のDD・契約書データとの紐づけ
- **Step3**：DD文書の文脈を読み取り、関連条文を自律的に判断して提示する
  （AIエージェント的な振る舞いへの拡張）

なお、今回は対象法令を宅地建物取引業法・金融商品取引法の一部条文（計8件）に絞っている。
e-Gov法令APIには各法令の全条文を取得する手段があり、技術的には全条文対応も可能だが、
今回のゴールは「構想の一部分（条文IDで紐づけた検索とAPI連携の仕組み）をPHPで動く形にする」
ことであり、全条文の網羅やタグ付け方針の精緻化は、自社データと統合する段階（Step2以降）で
体系的に検討する方が適切と判断し、今回のスコープからは意図的に外している。

## 利用データについて

法令データは e-Gov法令検索（デジタル庁）が提供する公開情報を参照しています。
e-Govのデータは「政府標準利用規約（第2.0版）」に基づいて利用可能です。
本READMEおよびコード中のサンプルデータは学習・課題提出目的での利用を想定しています。
